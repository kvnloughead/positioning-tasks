As soon as we apply top: 0 the parent element jumps to the top of the page! Why would that be? Well, recall that phrase from the previous lesson â€” "closest positioned ancestor". An ancestor is positioned if it has a non-default position, so the immediate parent of the blue container is not positioned.

When this happens, the browser checks the next containing element to see if it's positioned, and so on. If you use devtools to check each successive parent element of the blue box you'll note that none of its ancestors are positioned. When this happens, the container will be positioned with respect the the browser window.

Answer First, we need to give the grandparent element a non-default position. We'll use position: relative for this, since it won't produce any unwanted side effects. Then we can position the blue container absolutely and give it left: 0. We can omit top: 0, since it is implied by default. We can position the first child absolutely by giving it a negative top value equal to its height. The second child can be positioned with bottom: 0 and right: 0 Note that for the first child, it isn't any easier for us to position the element absolutely compared to doing so relatively. Each have their places.
